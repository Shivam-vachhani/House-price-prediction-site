## рџљЂ Getting Started

### Prerequisites

- Python 3.9+
- Node.js 18+
- npm or yarn

---

### 1. Clone the repository

```bash
git clone https://github.com/Shivam-vachhani/NivasIQ.git
cd NivasIQ
```

---

### 2. Backend setup

```bash
cd backend
python -m venv venv
source venv/bin/activate        # Windows: venv\Scripts\activate
pip install -r requirements.txt
```

**Run the Flask server:**

```bash
python app.py
# or with gunicorn for production:
gunicorn app:app
```

The API will be available at `http://127.0.0.1:5000` or `http://192.168.x.x:5000` depending on your local IP address.

---

### 3. Generate market stats (first time only)

Open `notebooks/NivasIQ_Training.ipynb` in Jupyter and run all cells. This will:

- Train/load the XGBoost model
- Generate `backend/market_stats.json`
- Generate `backend/model_comparison.json`

```bash
cd notebooks
jupyter notebook NivasIQ_Training.ipynb
```

---

### 4. Frontend setup

```bash
cd frontend
npm install
```

Create a `.env` file in the `frontend/` directory:

```env
VITE_AREA_URL=http://127.0.0.1:5000/areas
VITE_PREDICTION_http://127.0.0.1:5000/predict
VITE_MARKET_TRENDS_API=http://127.0.0.1:5000/market-stats
VITE_MODEL_COMPARISON_API=http://127.0.0.1:5000/model-comparison

```

**Run the dev server:**

```bash
npm run dev
```

Open `http://localhost:5173` in your browser.

## рџ”Њ API Endpoints

| Method | Endpoint            | Description                            |
| ------ | ------------------- | -------------------------------------- |
| `GET`  | `/areas`            | Avlaible area                          |
| `POST` | `/predict`          | Predict flat price                     |
| `GET`  | `/market-stats`     | Market analytics data                  |
| `GET`  | `/model-comparison` | Model comparison metrics + predictions |

### POST `/predict`

**Request body:**

```json
{
  "area": "bodakdev",
  "bhk": 2,
  "furnishing": "unfurnished",
  "area_sqft": 500,
  "is_new_property": 0,
  "floor": 8,
  "total_floors": 10
}
```

**Response:**

```json
{
  "area_encoded": 8875.59,
  "predicted_price": 50.66607,
  "relative_floor": 0.8,
  "status": "success"
}
```

> `area_encoded` is in **Thousends**. It's a avrage price per sqft in perticular area.
> `predicted_price` is in **lakhs**. Multiply by `100000` for rupees.

## рџ§  Model Details

### Features used

| Feature                | Description                |
| ---------------------- | -------------------------- |
| `BHK`                  | Number of bedrooms (1вЂ“6)   |
| `Area_in_sqft`         | Carpet area in square feet |
| `Area_Encoded`         | Label-encoded locality     |
| `is_New_Property`      | New flat = 1, resale = 0   |
| `is_Furnished`         | Furnished = 1, else 0      |
| `Relative_Floor_Level` | floor / total_floors       |

### Feature importance

```
BHK                 52.8%  в–€в–€в–€в–€в–€в–€в–€в–€в–€в–€в–€в–€в–€в–€в–€в–€в–€в–€в–€в–€в–€в–€в–€в–€
Area in sqft        39.2%  в–€в–€в–€в–€в–€в–€в–€в–€в–€в–€в–€в–€в–€в–€в–€в–€в–€в–€в–€
Area Encoded         4.6%  в–€в–€
is New Property      2.4%  в–€
is Furnished         0.6%
Relative Floor       0.4%
```

BHK configuration + carpet area together explain **92%** of price variance.

### Why XGBoost over Random Forest?

Random Forest achieves a slightly higher test RВІ (91.3% vs 90.4%) but has an overfit gap of **0.061** вЂ” 4Г— larger than XGBoost's **0.015**. In production, a lower overfit gap means the model generalises more reliably to property types it hasn't seen during training.

**Balanced score** (60% accuracy + 40% generalisation):

| Model             | Score       |
| ----------------- | ----------- |
| XGBoost           | **0.942** вњ“ |
| Random Forest     | 0.864       |
| Linear Regression | 0.831       |
